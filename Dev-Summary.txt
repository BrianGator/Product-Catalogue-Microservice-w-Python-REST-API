# Product Catalogue Microservice - Developer Summary

## Application Overview
This is a Python-based microservice for an eCommerce Product Catalogue. It provides a RESTful API to manage products (CRUD operations) and includes a web-based administrative interface for manual interaction. The service is built using modern DevOps practices, including Test-Driven Development (TDD) and Behavior-Driven Development (BDD).

## Technologies Used
- **Backend**: Python 3.8+, Flask (Web Framework)
- **Database**: SQLAlchemy (ORM), SQLite (Relational Database)
- **Testing (TDD)**: unittest, coverage.py
- **Testing (BDD)**: Behave (Gherkin syntax), Selenium (Web automation)
- **Mocking**: Factory Boy (Fake data generation)
- **Frontend**: HTML5, Bootstrap 5 (Styling), jQuery (AJAX interaction)

## Project Structure
- `/service/`: Core application logic.
    - `models.py`: Database schema and ORM methods (Product, Category).
    - `routes.py`: REST API endpoints and web routing.
    - `__init__.py`: Application factory and initialization.
    - `static/`: Frontend assets (Admin UI).
- `/tests/`: Unit tests and Integration tests.
    - `test_models.py`: Validates database logic and validation rules.
    - `test_routes.py`: Validates API endpoint behavior and HTTP status codes.
    - `factories.py`: Definitions for generating random test data.
- `/features/`: BDD testing environment.
    - `products.feature`: Gherkin scenarios for user stories.
    - `steps/`: Python implementations of the Gherkin steps.
    - `environment.py`: Setup for the Selenium browser and server context.

## Key Programming Concepts
- **RESTful API Design**: Clean URL structures and appropriate use of HTTP methods (GET, POST, PUT, DELETE).
- **ORM (Object-Relational Mapping)**: Mapping Python classes to database tables to avoid writing raw SQL.
- **Model-View-Controller (MVC)**: Separation of data (models), logic (routes/controllers), and presentation (static UI).
- **Graceful Error Handling**: Custom error handlers that return JSON responses for API failures.
- **Automated Testing**: Using mocking and sequences to ensure code reliability without manual verification.
